# PMP Exam Master Brief: Manage Communications (Process · Task 2)

---

## 1. Topic Snapshot — Plain English

Managing communications is the disciplined practice of planning, executing, and monitoring information flow so the right information reaches the right stakeholders at the right time, in the right format. PMI distinguishes communication (information transfer) from engagement (active collaboration). Topics include communication management plan, communication methods (push/pull/interactive), 5C's of effective communication, channels formula, and continuous monitoring of effectiveness. The Process domain is 50% of the exam; communications threads into stakeholder, risk, and integration topics. PMs spend ~90% of their time communicating. In real projects, communication plans are written and ignored; on the exam, PMI rewards the option that tailors communication to audience, monitors effectiveness, and adjusts continuously.

**In one sentence:** This topic is about deliberate, tailored, monitored information flow — not status reports.

---

## 2. The PMI Mindset on This Topic

### Predictive vs Agile vs Hybrid Lens

| Lens | Communication Management Looks Like |
|---|---|
| **Predictive** | Formal communication management plan; scheduled reports; documented distribution |
| **Agile** | Information radiators; daily ceremonies; high-bandwidth conversation; written summaries minimal |
| **Hybrid** | Communication plan + information radiators + iteration ceremonies |

### Top 5 PMI-isms on This Topic

1. **Tailor to audience.** Same information, different format and depth.
2. **Push, Pull, Interactive — pick deliberately.** Match method to need.
3. **Monitor effectiveness, not just delivery.** Sent ≠ received.
4. **Information radiator > scheduled report when possible.** Always-available beats periodic.
5. **5C's: Correct, Concise, Clear, Coherent, Complete.** Skip Concise to your peril.

### Ethics Angle

- **Honesty:** Communicate true status; never optimism theater.
- **Fairness:** All stakeholders informed equally per legitimate need.
- **Respect:** Communication format respects audience preference.
- **Responsibility:** Communication breakdowns are PM's to surface and fix.

## 4. Key Terms, Formulas & Acronyms

| Term / Formula | Definition / Calculation | Exam Trap to Avoid |
|---|---|---|
| **Communication Management Plan** | Documents what, when, how, to whom | Trap: writing then ignoring |
| **Push Communication** | Sender pushes to recipient (email, report) | Trap: assuming push = received |
| **Pull Communication** | Recipient retrieves on demand (portal, wiki) | Trap: assuming all pull |
| **Interactive Communication** | Real-time multi-direction (meeting, call) | Trap: defaulting to interactive |
| **5C's of Effective Communication** | Correct, Concise, Clear, Coherent, Complete | Trap: skipping Concise |
| **Channels Formula** | n(n-1)/2 where n = participants | Trap: arithmetic error |
| **Information Radiator** | Always-visible status display | Trap: behind-login dashboard |
| **Feedback Loop** | Confirming message received and understood | Trap: assuming silence = received |
| **Encoding / Decoding** | Sender packages; receiver interprets | Trap: assuming same code |
| **Noise** | Interference between sender and receiver | Trap: ignoring noise sources |
| **Communication Effectiveness** | Measure: was message received + understood + actioned | Trap: measuring only delivery |
| **Stakeholder Communication Requirements** | Per-stakeholder format/frequency/content | Trap: one-size-fits-all |

## 5. PM Behavior Cheat Sheet — FIRST / NEXT / BEST / EXCEPT

| Stem Signal | PM Should FIRST Do | Because |
|---|---|---|
| Stakeholder says **"not informed"** | Acknowledge; check communication plan for gaps; update plan | Engagement design |
| Communication plan **outdated** | Update through change request; reflect current audience needs | Living document |
| **Status report** ignored by stakeholders | Investigate format/channel/relevance; tailor | Sent ≠ received |
| Stakeholder requests **more frequent updates** | Accommodate; update communication plan | Responsiveness |
| **6+ people** added to project | Channels formula: communication complexity rises non-linearly | Plan additional channels |
| Real-time visibility **available** | Use information radiator; reduce push reports | Pull > push when possible |
| **Critical announcement** to make | Interactive (in-person/call) primary; written confirmation after | High-stakes = interactive |
| **Routine** weekly status | Push or pull as audience prefers | Routine = push or pull |
| Stakeholder **misinterprets** message | Add feedback loop (paraphrase, confirm); adjust encoding | Decoding gap |
| **Communication breakdown** identified | Root cause; update plan; communicate fix | Process improvement |

**Escalation-Specific Scenarios:**

| Stem Signal | Correct Escalation Logic |
|---|---|
| Communication breakdown affects critical stakeholder relationship | Brief sponsor with the gap, impact, and updated plan; engage stakeholder directly |
| Sponsor instructs communication suppression of material information | Refuse; document; escalate to next governance level if sponsor persists |

## 6. Visual Memory Aid

### Communication Method Selection

```text
INFORMATION TO COMMUNICATE
        │
        ▼
Urgency + sensitivity high?
   YES → INTERACTIVE (in-person, call) — high bandwidth
        + written follow-up
   NO  ↓
        ▼
Recipient needs control of timing?
   YES → PULL (portal, wiki, dashboard) — async
   NO  ↓
        ▼
Bulk routine information?
   YES → PUSH (email, report) — efficient
   NO  ↓
        ▼
Continuous status?
   YES → INFORMATION RADIATOR — always available
```

### Communication Mnemonic: **5C-PIM**

> **5C** = Correct · Concise · Clear · Coherent · Complete
> **P** = Push (sender drives)
> **I** = Interactive (real-time)
> **M** = Mediated/Pull (recipient drives)

Memory hook: **"Five C's, three channels — pick deliberately."**

### Channels Formula

> **n(n-1)/2** = number of communication channels among n participants
> Example: 6 people → 6×5/2 = 15 channels

## 7. Tailoring Considerations

### Predictive Project

- Formal communication management plan
- Scheduled reports
- Documented distribution list

### Agile / Hybrid

- Information radiators primary
- Daily ceremonies as standing communication
- Written summaries minimal

### Industry / Regulatory Factors

- **Banking / Capital Markets:** BSEC, Bangladesh Bank require formal documented communication
- Regulatory communication formats often prescribed
- **Multinational:** language + cultural norms vary
- **Public-sector:** communication often subject to records retention

## 8. Common Wrong Answers & Why PMI Rejects Them

| Tempting Answer | Why Experienced PMs Pick It | Why PMI Marks It Wrong |
|---|---|---|
| "Send everything to everyone" | Inclusive | Noise; ignored |
| "Same format for all audiences" | Consistency | Misses audience needs |
| "Assume silence = agreement" | Pragmatism | Decoding gap likely |
| "Push only; pull is passive" | Active | Mismatched method |
| "Skip feedback loop; trust delivery" | Efficiency | Sent ≠ received |
| "Update plan at closure" | Lessons | Living document |
| "All communication via email" | Documented | Format mismatch |

## 9. 10 Hardest Scenario-Based Questions

---

**Q1 — Ethics Dilemma (Predictive)**

Sponsor instructs you to delay communicating a material risk to the board until after their performance review. The risk has just materialized. What should you do FIRST?

A. Delay as instructed.
B. Refuse; explain that material information must be communicated honestly and timely; brief sponsor on the trade-off and propose joint communication.
C. Delay and document.
D. Communicate directly to the board bypassing sponsor.

---

**Q2 — Agile Ceremony / Engagement**

The team uses a live kanban accessible to all. The sponsor still demands daily written status emails. Team finds emails distracting. What is the BEST action?

A. Continue daily emails.
B. Discontinue daily emails; coach sponsor on using the radiator; produce a brief weekly summary for the sponsor with trend changes; keep radiator live for everyone.
C. Discontinue all communication.
D. Have the team generate emails after hours.

---

**Q3 — FIRST Action, Conflict (Hybrid)**

Two stakeholders received conflicting information about scope — one from team, one from vendor. They now disagree on commitments. What should you do FIRST?

A. Choose one source.
B. Convene joint meeting with both stakeholders and vendor; establish single source of truth; update communication plan to prevent recurrence.
C. Escalate to legal.
D. Suspend communication until resolved.

---

**Q4 — Two Correct, Pick Best (Predictive)**

A new stakeholder is identified mid-project. Which TWO actions FIRST?

A. Add to stakeholder register with classification.
B. Send standard status report to new stakeholder.
C. Assess communication requirements (format, frequency, content) for new stakeholder.
D. Wait until next planning cycle.
E. Escalate to sponsor.

---

**Q5 — Ordered Sequence (Predictive)**

You are creating a communication management plan. Arrange:

1. Define methods (push/pull/interactive) per audience.
2. Identify stakeholders and their information needs.
3. Schedule frequency and review communication effectiveness cadence.
4. Document plan; communicate plan to stakeholders.

A. 1 → 2 → 3 → 4
B. 2 → 1 → 3 → 4
C. 3 → 2 → 1 → 4
D. 4 → 1 → 2 → 3

---

**Q6 — EXCEPT Question (Predictive)**

Which of the following is NOT one of the 5C's of effective communication?

A. Correct
B. Concise
C. Clear
D. Coherent
E. Confidential

---

**Q7 — Ethics / Honesty Hardline**

You discover you have been sending an inaccurate status (CPI = 0.85 reported as 0.95) for two weeks. Sponsor presented to board with the inaccurate data. What do you do?

A. Continue sending inaccurate data to avoid embarrassment.
B. Immediately correct the data; communicate the correction to sponsor and board with apology and root cause; update communication process to prevent recurrence.
C. Send corrected data without explanation.
D. Hide the error.

---

**Q8 — Communication Strategy (Hybrid)**

Project has stakeholders across regulator (formal monthly), sponsor (weekly summary), and business (real-time). What is the BEST approach?

A. Send the same monthly report to all.
B. Tailor: monthly formal report for regulator; weekly summary for sponsor; live dashboard for business — single source, multiple formats.
C. Discontinue formal reporting.
D. Send daily detail to all.

---

**Q9 — Escalation Judgment (Hybrid)**

Sponsor has repeatedly instructed you to suppress mention of an open risk in stakeholder communications. The risk is material. You have engaged sponsor once with concern. What is the BEST next step?

A. Continue suppressing.
B. Brief sponsor again with the ethical and impact dimensions; if sponsor refuses, escalate to next governance level (PMO, audit, compliance) with documented instructions and material risk.
C. Communicate directly to all stakeholders bypassing sponsor.
D. Resign quietly.

---

**Q10 — Multiple Response (Agile/Hybrid)**

Stakeholder feedback indicates communication is "not landing." Multiple stakeholders report not understanding status. Which THREE actions MOST appropriate? (Select three.)

A. Investigate format/channel/content per audience.
B. Send more email.
C. Add feedback loops (paraphrase, confirm, ask questions).
D. Mandate stakeholders attend all meetings.
E. Audit 5C's adherence in current communications.
F. Switch all communication to formal documents.

## 10. Answer Key with Full Justification

---

**Q1 — Correct Answer: B**

**PMI Justification:** Honesty principle; material information cannot be delayed for political reasons.

**Mindset Lesson:** Material risk communication timely; sponsor pressure does not waive.

---

**Q2 — Correct Answer: B**

**PMI Justification:** Coach sponsor + brief weekly summary + radiator live.

**Mindset Lesson:** Push reduced to summary; radiator covers real-time.

---

**Q3 — Correct Answer: B**

**PMI Justification:** Single source of truth + plan update.

**Mindset Lesson:** Conflicting information requires joint reconciliation + plan update.

---

**Q4 — Correct Answers: A and C**

**PMI Justification:** Register + needs assessment.

**Mindset Lesson:** New stakeholder = register + assess communication requirements.

---

**Q5 — Correct Answer: B (2 → 1 → 3 → 4)**

**PMI Justification:** Audience first → methods → frequency → document.

**Mindset Lesson:** Audience-first communication planning.

---

**Q6 — Correct Answer: E**

**PMI Justification:** 5C's are Correct, Concise, Clear, Coherent, Complete. Confidential is not one.

**Mindset Lesson:** Memorize 5C's exactly.

---

**Q7 — Correct Answer: B**

**PMI Justification:** Honesty + Responsibility — correct, communicate, apologize, root cause.

**Mindset Lesson:** Inaccurate communication corrected openly with root cause.

---

**Q8 — Correct Answer: B**

**PMI Justification:** Tailoring single source, multiple formats.

**Mindset Lesson:** Hybrid communication tailoring is the answer for multi-audience.

---

**Q9 — Correct Answer: B**

**PMI Justification:** Persistent material suppression escalated to governance.

**Mindset Lesson:** Material risk suppression escalated to next governance level — never to all stakeholders without sponsor.

---

**Q10 — Correct Answers: A, C, E**

**PMI Justification:**

- A: Diagnose per audience.
- C: Feedback loops.
- E: 5C's audit.
- B: More email is more noise.
- D: Mandate doesn't equal understanding.
- F: Formal documents may worsen mismatch.

**Mindset Lesson:** Communication not landing = diagnose + feedback loops + 5C's. Never volume increase, mandates, or format escalation.

## 12. 60-Second Recap Card

**MANAGE COMMUNICATIONS — Exam Morning Card**

- Tailor to audience. Same info, different format/depth.
- Push, Pull, Interactive — pick deliberately.
- 5C's: Correct, Concise, Clear, Coherent, Complete.
- Channels formula: n(n-1)/2.
- Information radiator > scheduled push when possible.
- Monitor effectiveness, not just delivery. Sent ≠ received.
- Communication plan = living document.
- Material information communicated honestly + timely.
- New stakeholder = register + communication requirements.
- Communication breakdown = diagnose + feedback loops + 5C's.

**Escalation one-liner to burn in:** *PMI escalates communication suppression with documented instructions and material risk — never absorbs, never bypasses sponsor to stakeholders, never delays material disclosure for political timing.*
