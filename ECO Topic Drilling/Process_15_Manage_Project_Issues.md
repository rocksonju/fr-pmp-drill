# PMP Exam Master Brief: Manage Project Issues (Process · Task 15)

---

## 1. Topic Snapshot — Plain English

Managing project issues is the practice of identifying, logging, assigning ownership, root-causing, resolving, and communicating problems that have materialized — distinct from risks (which haven't yet happened) and impediments (which slow the team). PMI treats issue log as the second log alongside risk register, with strict separation: when a risk occurs, it migrates from risk register to issue log. Topics include issue identification, issue log, owner, priority, resolution, and escalation. The Process domain is 50% of the exam; issues thread into risk and integration. In real projects, issues accumulate without owners; on the exam, PMI rewards prompt logging + owner assignment + resolution discipline.

**In one sentence:** This topic is about logging, owning, resolving, and communicating issues — separate from risks and impediments, with discipline.

## 2. The PMI Mindset on This Topic

### Predictive vs Agile vs Hybrid Lens

| Lens | Issue Management Looks Like |
|---|---|
| **Predictive** | Formal issue log; weekly review; ownership tracked |
| **Agile** | Issue board visible; daily standup surfaces; SM/PM addresses |
| **Hybrid** | Issue log + agile ceremonies |

### Top 5 PMI-isms on This Topic

1. **Issue ≠ Risk ≠ Impediment.** Materialized vs potential vs slowing.
2. **Every issue has an owner.** Not "the team."
3. **Root cause before action.**
4. **Resolution timeboxed where possible.**
5. **Communicate resolution to affected parties.**

### Ethics Angle

- **Honesty:** Issues surfaced openly.
- **Responsibility:** PM owns issue process.
- **Fairness:** Priority on impact, not politics.
- **Respect:** Issue raisers heard.

## 4. Key Terms, Formulas & Acronyms

| Term / Formula | Definition / Calculation | Exam Trap to Avoid |
|---|---|---|
| **Issue** | Problem that has materialized | Trap: confusing with risk |
| **Issue Log** | Living document of issues | Trap: stale |
| **Owner** | Person responsible for resolution | Trap: unassigned |
| **Priority / Severity** | Impact-based ranking | Trap: arbitrary |
| **Root Cause Analysis** | Find underlying cause | Trap: symptom only |
| **5 Whys** | Sequential cause questioning | Trap: stopping early |
| **Resolution** | Issue closed with explanation | Trap: forgetting closure |
| **Workaround** | Temporary response | Trap: permanent workaround |
| **Issue Aging** | Time open | Trap: ignoring |
| **Issue Escalation** | Route beyond PM authority | Trap: undocumented |

## 5. PM Behavior Cheat Sheet — FIRST / NEXT / BEST / EXCEPT

| Stem Signal | PM Should FIRST Do | Because |
|---|---|---|
| **Issue raised** | Log + assign owner + priority + due date | Visible + owned |
| Issue **recurs** | Root cause (5 Whys); fix cause | Symptom failure |
| Issue **aging** | Review priority; escalate if needed | Aging signals |
| Multiple **similar** issues | Cluster; identify common cause | Pattern |
| Issue is actually a **risk** | Move to risk register | Distinct |
| Issue **exceeds PM authority** | Escalate to sponsor / governance | Authority |
| Issue **affects vendor** | Engage vendor through contract | Vendor coordination |
| **Stakeholder** raises issue | Log; assign owner; resolve; communicate | Stakeholder respect |
| Issue resolved | Close in log; communicate to affected | Closure |
| **Workaround** in place | Document; flag as temporary; address root cause | Don't normalize |

**Escalation-Specific Scenarios:**

| Stem Signal | Correct Escalation Logic |
|---|---|
| Issue exceeds PM authority | Brief sponsor with issue, impact, attempted resolution, recommendation |
| Issue requires cross-program decision | Engage program governance |

## 9. 10 Hardest Scenario-Based Questions

---

**Q1 — Ethics Dilemma (Predictive)**

Sponsor instructs you not to log a critical issue affecting board-presented status. The issue is real and has impact. What should you do FIRST?

A. Don't log.
B. Refuse; explain issue log integrity; log the issue; brief sponsor on the cost of suppressing.
C. Log with low priority.
D. Defer.

---

**Q2 — Agile Ceremony / Engagement**

Issue log has 40 open issues, all "high priority." Team confused. What is the BEST action?

A. Continue.
B. Reprioritize with stakeholders on impact basis; cluster similar issues; address top-priority first.
C. Close all old ones.
D. Send list to sponsor.

---

**Q3 — FIRST Action, Conflict (Hybrid)**

Stakeholder raised same issue three sprints in a row. Each time we said "we'll address." Stakeholder is frustrated. What should you do FIRST?

A. Apologize and continue.
B. Engage stakeholder; root cause why issue keeps surfacing without resolution; commit to specific resolution path; communicate progress.
C. Escalate stakeholder to sponsor.
D. Close issue without resolution.

---

**Q4 — Two Correct, Pick Best (Predictive)**

Issue log aging: 8 issues over 60 days old. Team morale dropping. Which TWO actions FIRST?

A. Prioritize aged issues by impact; clear or escalate top items.
B. Hide log.
C. Brief sponsor on aged issues requiring governance support.
D. Close all aged silently.
E. Add more issues for visibility.

---

**Q5 — Ordered Sequence (Agile)**

New issue raised. Arrange:

1. Engage owner for action; track to resolution.
2. Log: ID + description + owner + priority + due date.
3. Root cause analysis if recurring.
4. Close in log + communicate to affected.

A. 2 → 1 → 3 → 4
B. 1 → 2 → 3 → 4
C. 2 → 3 → 1 → 4
D. 3 → 1 → 2 → 4

---

**Q6 — EXCEPT Question (Predictive)**

Which is NOT typically a field in an issue log?

A. Issue ID
B. Description
C. Owner
D. Resolution
E. Project ROI

---

**Q7 — Ethics / Honesty Hardline**

You discover team has been silently working around a security issue by manually disabling a control. Possible compliance breach. What do you do?

A. Tell team to continue but document.
B. Stop workaround; brief sponsor + security/compliance; engage to address underlying issue; document past activities for assessment.
C. Quietly fix without disclosing.
D. Reprimand team.

---

**Q8 — Communication Strategy (Hybrid)**

Sponsor finds issue log "too noisy." Sponsor wants clean dashboard. What is the BEST approach?

A. Hide log.
B. Keep log live for team + PMO transparency; produce sponsor-tailored summary highlighting aged + escalated + closure velocity.
C. Stop logging.
D. Move to private list.

---

**Q9 — Escalation Judgment (Hybrid)**

Issue exceeds PM authority. Two engagement attempts with relevant authority. No movement. Cycle time impact significant. What is the BEST next step?

A. Continue absorbing.
B. Brief sponsor with issue, impact, attempts, recommended governance action.
C. Direct team to violate constraint.
D. Wait.

---

**Q10 — Multiple Response (Agile/Hybrid)**

Team delivering steadily but issues accumulating. Some aged 60+ days. Which THREE actions MOST appropriate? (Select three.)

A. Prioritize by impact + aging; clear top.
B. Close aged without resolution.
C. Brief sponsor on aged issues.
D. Mandate stop raising new.
E. Focused retro on issue patterns + structural causes.
F. Bonus for issue clearance.

## 10. Answer Key with Full Justification

---

**Q1 — Correct Answer: B**

**PMI Justification:** Issue log integrity + honest brief.

**Mindset Lesson:** Issue log = honesty record.

---

**Q2 — Correct Answer: B**

**PMI Justification:** Reprioritize + cluster + top-priority focus.

**Mindset Lesson:** All-high-priority = no prioritization.

---

**Q3 — Correct Answer: B**

**PMI Justification:** Root cause + commitment + progress.

**Mindset Lesson:** Recurring issue with same stakeholder needs root cause + commitment.

---

**Q4 — Correct Answers: A and C**

**PMI Justification:** Clear top + escalate aged.

**Mindset Lesson:** Aged log = prioritize + escalate.

---

**Q5 — Correct Answer: A (2 → 1 → 3 → 4)**

**PMI Justification:** Log → engage → root cause (if recurring) → close.

**Mindset Lesson:** Log first; root cause comes after pattern.

---

**Q6 — Correct Answer: E**

**PMI Justification:** Project ROI is portfolio metric, not issue log field.

**Mindset Lesson:** Issue log fields: ID, description, owner, priority, due, resolution, status.

---

**Q7 — Correct Answer: B**

**PMI Justification:** Stop + brief + engage + document.

**Mindset Lesson:** Compliance workaround escalated immediately.

---

**Q8 — Correct Answer: B**

**PMI Justification:** Transparency + tailored sponsor view.

**Mindset Lesson:** Sponsor discomfort with log = tailored view + coaching.

---

**Q9 — Correct Answer: B**

**PMI Justification:** Sponsor brief after exhausted attempts.

**Mindset Lesson:** Structural issues escalated with data.

---

**Q10 — Correct Answers: A, C, E**

**PMI Justification:**

- A: Prioritize and clear.
- C: Escalate aged.
- E: Focused retro on patterns.
- B: Fake closure.
- D: Mandate.
- F: Bonus produces gaming.

**Mindset Lesson:** Aged log = prioritize + escalate + retro on patterns.

## 12. 60-Second Recap Card

**MANAGE PROJECT ISSUES — Exam Morning Card**

- LORRC: Log · Own · Root cause · Resolve/escalate · Communicate closure.
- Issue ≠ Risk ≠ Impediment. Materialized vs potential vs slowing.
- Every issue has an owner.
- Risk occurs → migrate to issue log.
- Aging = priority signal.
- Recurring issue = root cause via 5 Whys.
- Workaround temporary; address root cause.
- Close in log + communicate to affected.
- Issue exceeds PM authority = escalate per governance.
- Compliance workaround escalated immediately.

**Escalation one-liner to burn in:** *PMI escalates issues with root cause, impact, attempts, and recommendation — never absorbs, never fake-closes, never hides compliance workarounds.*
